<objective>
Implement the HA config entry lifecycle in __init__.py and test it. async_setup_entry wires coordinator creation → first-refresh → runtime_data assignment. async_unload_entry returns True (HA clears runtime_data automatically). Tests cover D-07 case 4 (clean state after unload), the reload scenario, and the entry-level error states for auth failure and network failure — assertions that require hass.config_entries.async_setup() and therefore cannot live in test_coordinator.py (Plan 02).

Entry-level error state tests (auth failure, network failure from hass.config_entries.async_setup perspective) belong here because they exercise the full HA config entry machinery, not the coordinator in isolation.
Output: Fully implemented __init__.py and passing test_init.py covering setup, unload, reload, auth error, and network error.
