Create `README.md` at the repository root covering:

## Sections

### Header
- Title: "Gatus Integration for Home Assistant"
- One-line description: polls the Gatus monitoring API and exposes endpoint health as native HA sensor entities

### Installation
Two methods:
1. **HACS (recommended)**: Add custom repository → `https://github.com/akentner/homeassistant-gatus-integration` → Integration → Download
2. **Manual**: Copy `custom_components/gatus/` into HA config dir, restart HA

### Configuration
- Via HA UI: Settings → Integrations → Add → Gatus
- Fields:
  - **URL** — Base URL of your Gatus instance (e.g. `https://status.example.com`)
  - **API Key** (optional) — Bearer token if Gatus auth is enabled
  - **Entity prefix** (optional, default: `gatus_`) — Prefix for all entity object IDs
  - **Scan interval** (30–300 s, default: 60 s) — How often to poll (configurable via Options Flow after setup)

### Entities per Gatus endpoint
Table with columns: Entity, Type, Device Class, Unit, Description

| Entity | Type | Device Class | Unit | Description |
|--------|------|--------------|------|-------------|
| `{prefix}{key}_status` | Binary Sensor | connectivity | — | `on` = up, `off` = down |
| `{prefix}{key}_response_time` | Sensor | duration | ms | Last response time in milliseconds |
| `{prefix}{key}_uptime` | Sensor | — | % | 7-day uptime percentage |
| `{prefix}{key}_conditions` | Sensor | — | — | Condition results, e.g. `3/3` |

Binary sensor attributes: `last_check_timestamp`, `error_reason`, `response_duration_ms`, `consecutive_failures`

### Device Grouping
- Endpoints in the same Gatus group → one HA Device
- Device name = Gatus group name

### Requirements
- Home Assistant 2025.1.0+
- A running Gatus instance reachable from HA

Create `brand/` directory and download + resize the official Gatus logo to 256×256 px.

Steps:
1. `mkdir -p brand`
2. Download source: `https://github.com/TwiN/gatus/blob/master/web/static/logo-512x512.png?raw=true`
   - Use `curl -L -o brand/icon_source.png "https://github.com/TwiN/gatus/blob/master/web/static/logo-512x512.png?raw=true"`
3. Resize to 256×256 using Python with Pillow:
   ```python
   from PIL import Image
   img = Image.open("brand/icon_source.png").convert("RGBA")
   img = img.resize((256, 256), Image.LANCZOS)
   img.save("brand/icon.png", "PNG")
   ```
   If Pillow is not available, try ImageMagick: `convert brand/icon_source.png -resize 256x256 brand/icon.png`
   After success, remove the source file: `rm brand/icon_source.png`

If download fails (network), create a minimal 256×256 placeholder PNG using Python's built-in struct/zlib:
   ```python
   import struct, zlib
   def make_png(w, h):
       def chunk(t, d): return struct.pack('>I', len(d)) + t + d + struct.pack('>I', zlib.crc32(t+d)&0xffffffff)
       raw = b''.join(b'\x00' + bytes([100,149,237]*w) for _ in range(h))
       return b'\x89PNG\r\n\x1a\n' + chunk(b'IHDR', struct.pack('>IIBBBBB',w,h,8,2,0,0,0)) + chunk(b'IDAT', zlib.compress(raw)) + chunk(b'IEND', b'')
   open('brand/icon.png','wb').write(make_png(256,256))
   ```
